// UNDER THE HOOD

How Our Agents Work

Every agent we deploy follows a proven architecture. Here's what happens when you give an agent a task.

01

Task Received

User or system triggers a request — voice call, form submission, scheduled job, or another agent's handoff.

→
02

Orchestrator Routes

A master orchestrator agent analyses the task, determines complexity, and routes to the optimal specialist agent — or spawns multiple agents for parallel execution.

→
03

Agent Executes

The specialist agent loads its role definition, connects to relevant tools (CRM, database, calendar), retrieves context from its knowledge base via RAG, and executes autonomously.

→
04

Result Delivered

Task completed, audit trail logged, result delivered to user or handed to the next agent in the chain. Humans notified for review when guardrails require it.

// THE REAL POWER

Agent Swarms

A single AI agent is useful. A coordinated swarm of specialist agents working in parallel? That's transformational. This is the core of what we build.

ORCHESTRATOR

When a complex task arrives, our orchestrator doesn't hand it to one agent. It decomposes the task, spins up 5, 10, or 20 specialist agents simultaneously, each tackling a piece of the puzzle. They work in parallel — a researcher gathers data while a writer drafts copy while an analyst crunches numbers while a compliance agent reviews everything. Work that takes a team of humans a week gets done in minutes.

How a Swarm Executes

1

Task Decomposition

Complex request comes in. The orchestrator breaks it into independent subtasks — what can run in parallel? What has dependencies?

⟶
2

Agent Spawning

Specialist agents are created on-the-fly. Each gets a role, knowledge base, tools, and the optimal AI model for its task. A researcher gets Gemini Flash. A coder gets Claude Sonnet. A writer gets GPT-5.2.

⟶
3

Parallel Execution

All agents work simultaneously. They can communicate, share findings, and hand off work to each other. An agent that finishes early can spawn sub-agents for deeper analysis.

⟶
4

Consolidation

The orchestrator collects all results, resolves conflicts, synthesises findings, and delivers a unified output — reviewed by a quality gate agent before delivery.

// ANATOMY OF AN AGENT

What Every Agent Gets

Every agent we spawn — whether it lives for 5 minutes or runs permanently — is configured with these capabilities.

Role Definition

A specialised identity and mission: "You are a senior sales analyst at [Company]. Your job is to score leads and recommend actions." Not a generic assistant — a domain expert.

Knowledge Base (RAG)

Connected to your documents — SOPs, product specs, medical protocols, compliance rules, training materials. The agent doesn't guess — it retrieves authoritative answers from your data.

Tool Access

CRM, ERP, databases, calendars, email, voice calls, file storage, APIs, spreadsheets. Agents don't just talk — they log into your systems and do the work.

Persistent Memory

Remembers past interactions, decisions, and context across sessions. Your HR agent knows it approved John's leave last Tuesday and that his balance is now 12 days.

Guardrails & Escalation

Compliance rules, spending limits, approval gates, and human escalation triggers. The agent knows exactly when to act and when to ask a human.

Optimal AI Model

Each agent gets the best model for its task. A researcher gets Gemini (fast, cheap). A strategist gets Claude Opus (deep reasoning). Sensitive tasks get Llama 3 (on-premise).

// ENTERPRISE SECURITY

Guardrails & Governance

Data Sovereignty

Sensitive data processed on-premise via DGX Spark. Patient records, financials, and HR data never leave your building.

Human-in-the-Loop

Critical decisions require human approval. Agents escalate — they don't override. You define the escalation triggers.

Full Audit Trail

Every agent action is logged: what it did, why, which data it accessed, which model it used. Complete accountability.

Role-Based Access

Agents only access what they need. A sales agent can't see HR records. A marketing agent can't access financials.

Data Classification

Automatic routing: sensitive data stays on-premise, non-sensitive tasks use cloud models for best-in-class capability.

PDPA Compliant

Built for Malaysian data protection law from day one. Consent tracking, data residency, right-to-deletion — all handled.
