Frequently Asked Questions

Which cloud platforms does NovaGenAI support?

NovaGenAI supports all major cloud platforms including Amazon Web Services (AWS), Google Cloud Platform (GCP), and Microsoft Azure. We design multi-cloud and hybrid architectures that leverage the strengths of each platform while avoiding vendor lock-in.

How does AI assist with cloud migration?

Our AI agents automate the most complex and error-prone parts of cloud migration: dependency mapping, workload assessment, configuration generation, data migration validation, security policy translation, and post-migration performance monitoring. This reduces migration timelines by 40-60% and dramatically lowers risk.

Is my data PDPA-compliant during and after cloud migration?

Absolutely. Data sovereignty is central to every migration we design. We ensure all personal data governed by Malaysia's Personal Data Protection Act (PDPA) remains within compliant jurisdictions. For sensitive workloads, we deploy hybrid architectures where regulated data stays on-premise while non-sensitive workloads scale in the cloud.

What is a hybrid cloud architecture?

A hybrid cloud architecture combines on-premise infrastructure with public cloud services. Your most sensitive data and workloads remain on your own servers (or NVIDIA DGX Spark), while scalable, non-sensitive workloads run in the cloud. NovaGenAI designs the connectivity, security, and orchestration layer that makes this seamless.

How does NovaGenAI optimise cloud costs?

Our AI-powered FinOps agents continuously monitor your cloud spend, identify underutilised resources, recommend right-sizing, manage reserved instance purchasing, and automatically scale workloads based on demand patterns. Clients typically see 25-40% cost reduction within the first quarter after migration.
